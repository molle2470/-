# 소싱(상품 수집) 통합 시스템 설계서

## 1. 개요

국내 무재고 구매대행 셀러를 위한 통합 솔루션.
소싱처(공식몰/종합몰)에서 웹 크롤링으로 상품을 수집하고, 마진율/마켓수수료 기반으로 판매가를 자동 계산하여 마켓에 등록하며, 등록된 상품의 가격/재고 변동을 실시간 모니터링하여 마켓에 자동 반영하는 시스템.

## 2. 단계별 구현 계획

### Phase 1 (테스트)
- 소싱처: **무신사** 1개
- 판매 마켓: **네이버 스마트스토어** 1개
- 상품 수: **10~20개**
- 목표: 전체 플로우 동작 검증

### Phase 2 (확장)
- 소싱처 3~5개, 마켓 3~4개, 1,000~5,000개 상품
- Celery + 프록시 도입

### Phase 3 (본격 운영)
- 전체 소싱처/마켓, 5만개 이상
- DB 파티셔닝 + 인프라 최적화

## 3. 핵심 요구사항

### 3.1 소싱(상품 수집)
- **수집 방식:** 소싱처별 전용 웹 크롤러 (BaseCrawler 상속 구조)
- **입력:** 소싱처 카테고리 페이지 URL
- **브랜드 지정 필수:** 지재권 이슈로 다른 브랜드 상품 혼입 절대 방지
- **수집 갯수 제한:** 마켓별 등록 한도가 있으므로 사용자가 갯수 설정
- **수집 데이터:** 상품명, 가격(원가), 썸네일+대표사진 1-2장, 옵션(필수), 카테고리, 브랜드명, 재고/품절 여부
- **상세페이지:** 수집 대상 아님

### 3.2 소싱 플로우
```
소싱처 선택 + 카테고리 URL 입력
→ 브랜드 선택 (체크박스 + 수집 가능 갯수 표시)
→ 카테고리 선택 (체크박스 + 수집 가능 갯수 표시)
→ 수집 갯수 설정
→ 수집 시작
```

### 3.3 가격 설정 (3단계 마켓 템플릿)

#### 1단계: 공통 기본값 (한 번만 설정)
- 출고지, 반품/교환지, 택배사, A/S 번호, 원산지
- KC인증 정보 (구매대행/KC안전관리대상 아님)

#### 2단계: 마켓별 템플릿 (마켓 고유 설정, 공통값 상속/오버라이드)
- 마켓수수료 (스마트스토어 8%, 지마켓/옥션 20%, 쿠팡 15% 등)
- 배송비 설정, 제주/도서산간 추가비
- 반품 배송비
- 상품명 길이 제한
- 마켓별 고유 옵션 (쿠팡: 출고 소요일, 묶음배송 등)
- 포인트/할인 설정
- 상품정보제공고시 템플릿

#### 3단계: 가격 자동 계산
```
판매가 = 원가 ÷ (1 - 마켓수수료 - 마진율)

예: 원가 50,000원, 마진율 20%
  스마트스토어: 50,000 ÷ (1 - 0.08 - 0.20) = 69,444원
  쿠팡:        50,000 ÷ (1 - 0.15 - 0.20) = 76,923원
```

### 3.4 마켓별 SEO 자동 생성

각 마켓별 규칙/패턴을 미리 설정하면 등록 시 자동 채움:

#### 스마트스토어
- 태그 자동 생성 (브랜드 + 카테고리 키워드 + 상품명 키워드, 최대 10개)
- Page Title 패턴: "{브랜드} {상품명} {키워드}"
- Meta Description 패턴
- 모델명 (카탈로그 연결용)
- 상품속성 (카테고리별 자동 매핑)

#### 쿠팡
- **카탈로그 매칭 절대 금지** (단독 상품으로만 등록)
- 검색 키워드 자동 생성

#### 11번가 / 롯데온
- 검색 태그/키워드 자동 생성

#### 공통 자동 채움
- 브랜드명 → 소싱 데이터
- 카테고리/상품속성 → 카테고리 매핑
- 원산지, KC인증 → 마켓 템플릿 기본값
- 상품정보제공고시 → 카테고리별 템플릿

### 3.5 가격/재고 모니터링 + 자동 반영 (핵심 기능)

#### 모니터링 대상
- **마켓에 등록된 상품만** (수집만 한 상품은 제외)

#### 변동 자동 반영 트리거 3가지
```
1. 소싱처 원가 변동 감지
   → 판매가 자동 재계산 → 마켓 API로 가격 수정

2. 소싱처 재고 변동 감지
   → 품절: 마켓에서 품절 처리
   → 재입고: 마켓에서 재활성화

3. 마진율 수동 변경
   → 해당 마켓 전체 상품 판매가 재계산 → 마켓 API로 일괄 수정
```

#### 스마트 주기
- 인기/재고 적은 상품: 10~15분 (±2분 랜덤)
- 일반 상품: 30분~1시간 (±5분 랜덤)

#### 봇 감지 회피
- 요청 간 랜덤 딜레이, User-Agent 로테이션, 소싱처별 요청 분산

### 3.6 에러 처리
- 크롤링 실패 시 자동 재시도 (3회)
- 지속 실패 시 해당 상품 자동 품절 처리 (안전 조치)
- 품절 처리된 상품이 이후 크롤링 성공 시 → 자동 재활성화
- 대시보드 에러 표시

### 3.7 카테고리 자동 매핑
- 소싱처 카테고리 → 마켓 카테고리 자동 매핑

### 3.8 마켓 등록

#### 등록 플로우
```
수집된 상품 선택
→ 마켓그룹 선택 (사업자 선택)
  ○ 전체  ○ 사업자A  ○ 사업자B  ...
→ 마켓 구분 (어떤 마켓에 올릴지)
  ☑ 스마트스토어  ☑ 옥션  ☐ 쿠팡  ...
→ 마켓 계정 선택 (실제 계정)
  ☑ 사업자A-스마트스토어
  ☑ 사업자A-옥션
  ⛔ 사업자A-쿠팡 (소명미완료 → 비활성)
→ 업로드
```

#### 마켓별 고정 이미지
- 상단/하단 고정 이미지를 마켓별로 미리 등록
- **상세페이지 구성:** 고정 상단 → 대표이미지1(썸네일) → 대표이미지2 → 고정 하단

#### 쿠팡 브랜드 소명 관리
- **쿠팡만** 브랜드 소명이 필요 (나머지 마켓은 자유 등록)
- 소명 유형: 지재권 브랜드 / 가품의심 유통경로
- **쿠팡 계정 4개** × 브랜드별 개별 소명 관리
- 소명 미완료 브랜드는 해당 쿠팡 계정에서 등록 자동 차단

```
[쿠팡 소명 관리 화면]
┌──────────┬────────────┬──────────┬──────────┬──────────┬──────────┐
│ 브랜드    │ 소명 유형   │ 쿠팡계정1 │ 쿠팡계정2 │ 쿠팡계정3 │ 쿠팡계정4 │
│ 나이키   │ 지재권      │ ✅       │ ❌       │ ✅       │ ❌       │
│ 구찌     │ 가품의심    │ ✅       │ ✅       │ ❌       │ ✅       │
└──────────┴────────────┴──────────┴──────────┴──────────┴──────────┘
```

## 4. 시스템 아키텍처

```
[Next.js 프론트엔드 (대시보드)]
        ↕ API 호출
[FastAPI 백엔드]
   ├── 크롤링 엔진 (소싱처별 전용 크롤러)
   ├── 모니터링 스케줄러 (스마트 주기)
   ├── 가격 계산 서비스
   ├── 상품 관리 서비스
   ├── 카테고리 매핑 서비스
   ├── SEO 자동 생성 서비스
   └── 마켓 등록/동기화 서비스
        ↕
[PostgreSQL DB]
```

### 4.1 크롤링 엔진 구조

```
[크롤러 매니저]
   ├── BaseCrawler (공통 인터페이스)
   │    ├── 요청 간 랜덤 딜레이
   │    ├── 자동 재시도 (3회)
   │    ├── 재시도 실패 시 상품 품절 처리
   │    └── User-Agent 로테이션
   │
   ├── MusinsaCrawler (무신사) ← Phase 1
   ├── SSGCrawler (SSG)
   ├── LotteONCrawler (롯데ON)
   ├── HyundaiCrawler (현대Hmall)
   └── ... (소싱처 추가 시 BaseCrawler 상속)
```

### 4.2 크롤링 흐름
1. 카테고리 페이지 접속
2. 브랜드 목록 + 상품 갯수 스캔 (체크박스 UI용)
3. 사용자 선택 (브랜드/카테고리/갯수)
4. 선택 조건에 맞는 상품 페이지 순회
5. 상품별 데이터 추출 (브랜드 일치 검증) → DB 저장
6. 마켓 등록된 상품만 모니터링 스케줄러에 등록

### 4.3 마켓 동기화 엔진

```
[마켓 어댑터 매니저]
   ├── BaseMarketAdapter (공통 인터페이스)
   │    ├── 상품 등록
   │    ├── 가격 수정
   │    ├── 재고/품절 처리
   │    └── SEO 필드 채움
   │
   ├── NaverAdapter (스마트스토어 Commerce API) ← Phase 1
   ├── CoupangAdapter (쿠팡 Wing API, 카탈로그 매칭 금지)
   ├── ElevenStAdapter (11번가 Open API)
   ├── GmarketAdapter (지마켓 API)
   ├── AuctionAdapter (옥션 API)
   └── LotteOnAdapter (롯데온 API)
```

### 4.4 백그라운드 작업
- **Phase 1:** APScheduler (10-20개 규모에 충분)
- **Phase 2+:** Celery + Redis (수천~수만 상품 분산 처리)
- 구조는 Phase 1부터 교체 가능하도록 설계

### 4.5 확장 시 필요 인프라 (Phase 2+)
- **Celery + Redis:** 크롤링 작업 분산 처리
- **프록시 서비스:** IP 로테이션으로 봇 감지 회피
- **DB 파티셔닝:** 변동 이력 테이블 월별 파티셔닝

## 5. 데이터 모델

### 5.1 소싱처 (Source)
- id, 이름, 베이스 URL, 크롤러 타입, 상태 (활성/비활성)

### 5.2 브랜드 - 마스터 (Brand)
- id, 브랜드명, 지재권 승인 여부

### 5.3 소싱처별 브랜드 매핑 (SourceBrand)
- id, brand_id (FK → Brand), source_id (FK → Source)
- 소싱처 내 표기명 (예: "NIKE", "나이키" 등 소싱처마다 다름)

### 5.4 사업자 그룹 (BusinessGroup)
- id, 그룹명

### 5.5 마켓 (Market)
- id, 마켓명 (쿠팡/스마트스토어/옥션/지마켓/11번가/롯데온)
- 고정 상단 이미지 URL, 고정 하단 이미지 URL

### 5.6 마켓 계정 (MarketAccount)
- id, business_group_id (FK → BusinessGroup), market_id (FK → Market)
- 계정 ID, API 인증 정보, 상태 (활성/비활성)

### 5.7 쿠팡 브랜드 소명 (CoupangBrandClearance)
- id, brand_id (FK → Brand), market_account_id (FK → MarketAccount, 쿠팡 계정)
- 소명 유형 (지재권/가품의심)
- 소명 상태 (완료/미완료/진행중)
- 소명 완료일

### 5.8 공통 템플릿 (CommonTemplate)
- id, 출고지, 반품/교환지, 택배사
- A/S 전화번호, A/S 안내문구
- 원산지, KC인증 정보

### 5.9 마켓 템플릿 (MarketTemplate)
- id, market_id (FK → Market), common_template_id (FK → CommonTemplate)
- 마켓수수료 (%), 마진율 (%)
- 배송비 설정, 제주/도서산간 추가비, 반품 배송비
- 상품명 길이 제한
- 할인율, 포인트 설정
- 마켓 고유 설정 (JSON) - 쿠팡 출고소요일, 옥션 복수구매할인 등
- 상품정보제공고시 템플릿
- 출고지/반품지 오버라이드 (NULL이면 공통값 사용)

### 5.10 SEO 규칙 (SeoRule)
- id, market_id (FK → Market)
- 태그 생성 규칙 (패턴)
- Page Title 패턴
- Meta Description 패턴
- 키워드 생성 규칙

### 5.11 상품 (Product)
- id, source_id (FK → Source), brand_id (FK → Brand)
- 상품명, 원가, 썸네일 URL, 대표사진 URLs
- 소싱처 상품 URL, 소싱처 고유 상품 ID
- 카테고리 (소싱처 원본), 매핑된 마켓 카테고리
- 재고 상태 (정상/품절)
- 모니터링 등급 (인기/일반)
- 상태 (수집됨/등록됨/비활성)
- 최종 갱신 시간
- UNIQUE 제약: (source_id, source_product_id)

### 5.12 상품 옵션 (ProductOption)
- 조합형 옵션 지원 (색상 × 사이즈)

```
ProductOptionGroup: id, product_id, group_name (색상, 사이즈)
ProductOptionValue: id, group_id, value (블랙, M 등)
ProductVariant: id, product_id, option_combination (JSON), 재고상태, 추가가격
```

### 5.13 가격/재고 변동 이력 (PriceStockHistory)
- id, product_id, 변동 유형 (가격변동/품절/재입고), 이전값, 변경값, 감지 시간
- 인덱스: (product_id, created_at DESC), (change_type, created_at DESC)
- Phase 3에서 월별 파티셔닝 + 아카이빙 정책 적용

### 5.14 크롤링 작업 (CrawlJob)
- id, source_id, brand_id, 소싱처 URL, 수집 갯수 설정
- 상태 (대기/진행중/완료/실패), 수집된 갯수, 에러 내용, 시작/완료 시간

### 5.15 마켓 등록 (MarketListing)
- id, product_id, market_account_id (FK → MarketAccount)
- 판매가 (자동 계산), 등록 상태, 마켓 상품 ID, 등록 시간
- SEO 데이터 (태그, 타이틀, 메타설명 - JSON)

### 5.16 카테고리 매핑 (CategoryMapping)
- id, source_id, 소싱처 카테고리, market_id, 마켓 카테고리 ID
- 매핑 신뢰도, 수동 확인 여부

### 5.17 알림 (Notification)
- id, 유형 (가격변동/품절/재입고/크롤링에러/마켓동기화에러)
- 관련 상품 ID, 메시지, 읽음 여부, 생성 시간

## 6. 대시보드 화면 구성

### 6.1 사이드바 메뉴
```
├── 대시보드 (홈)
├── 소싱 관리
│    ├── 새 소싱 작업
│    ├── 소싱처 관리
│    └── 브랜드 관리
├── 상품 관리
│    ├── 전체 상품 목록
│    └── 가격/재고 변동 내역
├── 마켓 관리
│    ├── 사업자/계정 관리
│    ├── 마켓 템플릿 설정
│    │    ├── 공통 기본값
│    │    └── 마켓별 설정 (수수료/배송/SEO 규칙)
│    ├── 마켓별 고정 이미지
│    ├── 쿠팡 소명 관리
│    └── 마켓 등록 현황
├── 모니터링
│    ├── 크롤링 현황
│    ├── 마켓 동기화 현황
│    └── 알림 센터
```

### 6.2 대시보드 홈
- 총 상품 수 / 활성 상품 수 / 품절 상품 수
- 오늘 가격 변동 건수 / 재고 변동 건수
- 크롤링 상태 요약 (정상/에러)
- 마켓 동기화 상태 요약
- 최근 변동 알림 목록

### 6.3 새 소싱 작업
1단계: 소싱처 선택 + 카테고리 URL 입력
2단계: 브랜드 선택 (체크박스 + 갯수 표시)
3단계: 카테고리 선택 (체크박스 + 갯수 표시)
4단계: 수집 갯수 설정 → 수집 시작

### 6.4 상품 목록
- 필터: 소싱처별 / 브랜드별 / 상태별 / 카테고리별
- 검색: 상품명, 브랜드명
- 일괄 작업: 선택 → 마켓 등록 / 상태변경 / 삭제 / 모니터링 등급 변경
- 각 상품: 썸네일, 상품명, 브랜드, 원가, 판매가(마켓별), 재고상태, 등록 마켓, 최종갱신

### 6.5 상품 등록 (마켓 업로드)
```
상품 선택 → 업로드 모달:
┌─ 마켓그룹 (사업자) ─────────────────┐
│  ○ 전체  ○ 사업자A  ○ 사업자B ...  │
├─ 마켓 구분 ────────────────────────-┤
│  ☑ 스마트스토어  ☑ 옥션  ☐ 쿠팡 ...│
├─ 마켓 계정 ────────────────────────-┤
│  ☑ 계정1 (스마트스토어)              │
│  ☑ 계정2 (옥션)                     │
│  ⛔ 계정3 (쿠팡-소명미완료) ← 비활성 │
├──────────────────────────────────---┤
│  [닫기]              [업로드]        │
└─────────────────────────────────────┘
```

### 6.6 마켓 템플릿 설정
- 공통 기본값 탭: 출고지, 반품지, 택배사, A/S 등
- 마켓별 탭: 수수료, 배송비, SEO 규칙, 마켓 고유 옵션
- 공통값 상속 표시 (오버라이드 가능)

### 6.7 쿠팡 소명 관리
- 쿠팡 계정 4개 × 브랜드별 소명 상태 매트릭스
- 소명 유형 표시 (지재권/가품의심)
- 체크박스로 소명 완료/미완료 토글

### 6.8 알림 센터
- 가격 변동 / 품절 / 재입고 / 크롤링 에러 / 마켓 동기화 에러
- 읽음/안읽음 표시

## 7. 기술 스택

- **백엔드:** FastAPI + Python 3.12 (async/await)
- **프론트엔드:** Next.js 15 + React 19 + TypeScript + Tailwind CSS
- **데이터베이스:** PostgreSQL + SQLModel
- **크롤링:** httpx + BeautifulSoup / Playwright (소싱처별 선택)
- **백그라운드 작업:** Phase 1은 APScheduler, Phase 2+는 Celery + Redis
- **이미지 저장:** AWS S3
- **실시간 진행상황:** SSE (Server-Sent Events)

## 8. 범위 제외 (별도 서브프로젝트)

- 주문 처리 시스템
- 송장 자동등록
- SMS 자동 발송 (알리고 API 활용 예정)
- AI 이미지 변환 (모델→마네킹, 각도 변경 등) - 외부 API 활용 예정

## 9. 주요 소싱처

- **Phase 1:** 무신사
- **이후:** SSG, 롯데ON, 현대Hmall, 브랜드 공식몰

## 10. 판매 마켓

- 쿠팡, 네이버 스마트스토어, 옥션, 지마켓, 11번가, 롯데온
